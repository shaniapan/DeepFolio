## 一、产品愿景

**一句话定位**：Reader 是一个将大模型能力深度嵌入阅读流程的 AI 辅助阅读工具，让你不再需要在书本和 Chatbot 之间来回切换——AI 就住在你的书里。

**核心理念**：**把书读厚，也把书读薄**

- **读厚**：注释、关联、追问、辩论——让一本书长出枝丫，深入理解内化
- **读薄**：提炼、归纳、结构化——把一本书浓缩成精华，形成可迁移的知识

**差异化**：传统阅读 App 止步于"读完"，Reader 的目标是帮助用户**真正理解，深度记忆，并能用起来**。

---

## 二、目标用户与使用场景

**主要用户**：有深度阅读需求的个人用户（知识工作者、终身学习者、希望书本知识深度冲刷大脑、希望书本知识转化为行动的人）

**三个典型场景**：

| 场景 | 用户行为 | Reader 的角色 |
|------|---------|--------------|
| 工具式阅读 | 带着问题快速读书，只读对我有用的 20% | AI 向导，帮你定向萃取 |
| 沉浸式阅读 | 安静深读，遇到问题随时求助 | AI 伴侣，随叫随到不打扰 |
| 知识沉淀 | 读完后想留下什么、用起来 | AI 结构师，帮你构建知识体系 |

## 四、两种阅读模式

### 4.1 模式选择入口

每次点击进入一本书时，展示模式选择界面：

- 显示书籍封面、书名、上次阅读位置
- 两个模式卡片：⚡ 工具式 / 🌊 沉浸式
- 默认选中上次该书使用的模式
- 底部提示："进度和笔记两种模式共享，随时可切换"
- 阅读界面顶部始终可一键切换模式，切换不丢失任何数据与对话记录

---

### 4.2 ⚡ 工具式阅读

**核心理念**：书是原料，问题是导向，输出是目标。

#### 进入流程

**Step 1 · 锚定主体性**

用户填写"本书阅读目标"：
- 提示语：*"这本书你想解决什么问题？你最关心书里的哪个方面？"*
- 自动预填：从**全局用户档案**中提取相关内容，用户可确认或修改
- 提交后：系统合并 `全局档案 + 本书目标` → 生成**阅读上下文**
- 阅读上下文在阅读过程中**随时可修改**（读着读着，目标会变）

**Step 2 · AI 生成专属阅读地图**

- AI 扫描**全书目录 + 全文**（后台异步，不阻塞进入阅读）
- 输出：① 重点阅读章节 ② 可略读章节 ③ 可跳过章节
- 地图呈现在右侧 AI 面板，可点击跳转对应章节

**Step 3 · 进入阅读**

- 阅读界面右侧 AI 面板处于"引导模式"
- 顶部显示当前建议阶段（可拆卸功能，默认开启，可在设置中关闭）

#### 五步工具集（自由调用，无强制顺序）

AI 会根据当前阅读位置智能推荐优先展示哪个工具，但用户可无视推荐自由调用。

**① 列 · 狩猎阅读**
- 悬浮菜单直接展示预生成内容（来自本地 DB，零 token 消耗）：
  - 🌉 跨学科概念联想（如"这与博弈论的囚徒困境有相似结构"）
  - 🔗 旧知识关联（"与您在《X》第N章的[概念]相关"）
- 可选操作：大白话转述 / 加笔记 / 高亮

**② 变 · 三方会谈**
- `批判此观点` — AI 扮演"顶尖反对专家"进行反驳
- `此观点的边界在哪？` — AI 列举观点失效的场景
- `作者为何这样认为？` — AI 分析作者论证逻辑

**③ 缝 · 知识缝合**
- 章节结束时右侧面板出现轻提示卡（用户可忽略）
- `费曼测试` — 用户语音/文字复述，AI 评估准确性并补充遗漏
- `与旧知识连接` — AI 分析两个概念的互补/矛盾/包含关系
- `生成本章知识卡片` — AI 提炼核心要点，存入知识库

**④ 用 · 行动转化**
- 可随时触发（读完部分章节即可，不必等全书）
- 输出：基于**阅读上下文**的个性化行动清单
- 格式：下周/本月可执行的 3-5 个具体步骤
- 可导出为 Markdown 文件

---

### 4.3 🌊 沉浸式阅读

**核心理念**：阅读体验优先，AI 是背景支持。

#### 进入体验
- 直接跳转至上次阅读位置，无引导步骤
- AI 静默带入全局档案上下文
- 右侧 AI 面板默认**收窄**（细条），不占主阅读区

#### 触发 AI 的三种方式

**方式 A · 划词悬浮菜单**

选中文字后浮出菜单，分两区展示：
```
┌─────────────────────────────────────────┐
│  💬 提问  |  ✏️ 高亮 ▼  |  📝 加笔记     │  ← 操作区
├─────────────────────────────────────────┤
│  🌉 这与凯利公式（投资领域）的逻辑相似...  │  ← 预生成区
│  🔗 见《思考快与慢》第8章"损失厌恶"      │    （本地读取，零成本）
└─────────────────────────────────────────┘
```

**方式 B · 快捷键**
- `Cmd + K` — 呼出完整 AI 对话框（带入当前段落作为上下文）
- `Cmd + H` — 快速高亮当前段落
- `Cmd + N` — 快速添加笔记

**方式 C · 展开右侧面板**
- 点击右侧细条 → 展开完整 AI 面板
- 所有工具平铺可用（同工具式，无阶段引导）

#### 系统适时提示（Nudge）

- 默认开启，可在设置中关闭
- 触发场景：读完一章 / 连续阅读 45 分钟
- **关闭方式**：用户继续下滑，或在界面任意位置点击（不自动消失）
- 示例："读完第3章了，要做个费曼测试吗？" / "已连续阅读45分钟，回顾一下高亮笔记？"

## 五、AI 能力体系

### 5.1 Token 成本策略：预生成批量 + 本地缓存

**核心原则：日常阅读期间零 AI 实时调用**

```
【首次打开一本书】— 一次性批量调用
  AI 处理全书 → 生成并存入本地 SQLite：
  ├── 全书段落标注（类型 + 内容）
  ├── 所有章节摘要
  ├── 跨学科联想（基于全书内容）
  └── 旧知识关联（基于用户知识库对比）

【日常阅读】— 零 AI 调用
  前端读取本地 DB → 右侧面板"实时"显示预生成内容
  用户感受到 AI 跟着阅读进度走，实际是本地缓存渲染

【用户主动交互】— 按需 on-demand 调用
  ├── 用户点击"提问"
  ├── 触发"批判模式"
  ├── 执行"费曼测试"
  └── 请求"生成知识卡片 / 行动清单"
```

### 5.2 段落标注系统

**四种类型**（优先级从高到低）：
- 🔑 蓝色 — 关键论点段落（全书核心主张）
- ❓ 橙色 — 疑难段落（语义密度高或逻辑跳跃）
- 💡 绿色 — 跨书关联点（与用户已读书籍共鸣）
- 🌉 紫色 — 跨学科联想点（可用其他领域类比）

**密度控制规则**（防止视觉噪音）：
- 每段最多显示 1 个角标，同段多种类型取优先级最高者
- 每章角标总数 ≤ 该章段落数的 20%
- 相邻标注段落之间至少间隔 1 段无标注

### 5.3 全局功能

| 功能 | 说明 | 入口 |
|------|------|------|
| 🗺️ 专属阅读地图 | 按用户目标为章节排列权重 | 顶部工具栏 |
| 📊 全书结构分析 | 论点树 / 思维导图 | 顶部工具栏 |
| 🔗 跨书知识图谱 | 用户已读书籍的概念关联网络 | 顶部工具栏 |
| 📖 读薄输出 | 全书一页纸核心提炼 | 顶部工具栏 |
| 📋 行动清单 | 个性化、可执行的行动步骤 | 工具式工具区 |
| 🧩 作者视角档案 | 作者核心世界观分析 | 全书视图 |

### 5.4 AI 模型层（统一 Adapter 架构）

支持模型（用户自配 API Key）：
- OpenAI（GPT-4o 等）
- Anthropic Claude
- DeepSeek
- Qwen（通义千问）
- 后续可扩展更多模型

## 七、数据架构

### 7.1 双轨存储

| 存储层 | 技术 | 用途 |
|--------|------|------|
| 结构化数据库 | SQLite（本地） | 书籍、进度、笔记、高亮、AI 生成内容、知识卡片 |
| Markdown 文件 | 本地文件系统 | 每本书对应 .md 笔记文件，数据完全可迁移 |

### 7.2 核心数据模型

```
UserProfile（全局用户档案）
  ├── identity: 我是谁、背景、职业
  └── interests: 长期关注的问题和目标

Book（书籍）
  ├── file: 原始文件（EPUB/PDF/TXT）
  ├── metadata: 书名、作者、封面
  ├── readingContext: 本书阅读目标（可随时修改）
  ├── progress: 当前阅读位置
  └── mode: 上次使用的阅读模式

Annotation（段落标注，预生成）
  ├── bookId + paragraphId
  ├── type: KEY | HARD | LINK | BRIDGE
  ├── content: 预生成的对应内容
  └── crossDisciplinary: 跨学科联想文本

Note（笔记 & 高亮）
  ├── bookId + position
  ├── type: highlight | note
  ├── color: yellow | blue | green | red
  └── content: 批注文字

KnowledgeCard（知识卡片）
  ├── sourceBook + sourceChapter
  ├── concept: 核心概念
  ├── summary: AI 提炼的要点
  └── connections: 关联的其他卡片 ID

AIConversation（对话记录）
  ├── bookId + sessionId
  ├── mode: instrumental | immersive
  └── messages: 对话历史（切换模式时保留）
```

### 7.3 Markdown 笔记结构

```
/reader-notes/
  └── {书名}-{作者}/
      ├── reading-context.md    # 阅读目标 & 用户档案
      ├── chapter-notes.md      # 各章笔记汇总
      ├── highlights.md         # 所有高亮
      ├── knowledge-cards.md    # 知识卡片
      └── action-list.md        # 生成的行动清单
```

## 八、技术架构

### 8.1 技术栈

| 层 | 技术 | 选型理由 |
|----|------|---------|
| 前端 | React + TypeScript + Vite | 生态成熟，热更新快，AI 库丰富 |
| 后端 | Node.js + Express | 轻量，完整文件系统访问，AI 流式支持 |
| 数据库 | SQLite（better-sqlite3） | 本地运行，无需独立服务 |
| EPUB 渲染 | epub.js | 成熟的 EPUB 渲染库 |
| PDF 渲染 | PDF.js | Mozilla 出品 |
| AI 调用 | 自建 Adapter 层 | 统一多模型接口 |

### 8.2 项目结构

```
reader/
├── packages/
│   ├── frontend/          # React 应用（Vite）
│   │   └── src/
│   │       ├── pages/     # 书库 / 阅读 / 全书视图 / 知识库 / 设置
│   │       ├── components/# 阅读器、AI面板、悬浮菜单、工具区...
│   │       └── stores/    # 状态管理（Zustand / Jotai）
│   └── backend/           # Node.js 服务
│       └── src/
│           ├── routes/    # API 路由
│           ├── services/
│           │   ├── ai/    # Adapter：OpenAI / Claude / DeepSeek / Qwen
│           │   ├── epub/  # EPUB 解析
│           │   ├── pdf/   # PDF 解析
│           │   └── markdown/ # Markdown 双写
│           └── db/        # SQLite 数据模型 & 迁移
├── docs/
│   └── superpowers/specs/ # 设计文档
└── README.md
```

### 8.3 本地启动

```bash
npm start     # 同时启动前端 + 后端，浏览器自动打开 http://localhost:3000
npm run dev   # 开发模式（前后端热更新）
```

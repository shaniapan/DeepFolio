## 一、产品愿景

**一句话定位**：Reader 是一个将大模型能力深度嵌入阅读流程的 AI 辅助阅读工具，让你不再需要在书本和 Chatbot 之间来回切换——AI 就住在你的书里。

**核心理念**：**把书读厚，也把书读薄**

- **读厚**：注释、关联、追问、辩论——让一本书长出枝丫，深入理解内化
- **读薄**：提炼、归纳、结构化——把一本书浓缩成精华，形成可迁移的知识

**差异化**：传统阅读 App 止步于"读完"，Reader 的目标是帮助用户**真正理解，深度记忆，并能用起来**。

---

## 二、目标用户与使用场景

**主要用户**：有深度阅读需求的个人用户（知识工作者、终身学习者、希望书本知识深度冲刷大脑、希望书本知识转化为行动的人）

**三个典型场景**：

| 场景 | 用户行为 | Reader 的角色 |
|------|---------|--------------|
| 工具式阅读 | 带着问题快速读书，只读对我有用的 20% | AI 向导，帮你定向萃取 |
| 沉浸式阅读 | 安静深读，遇到问题随时求助 | AI 伴侣，随叫随到不打扰 |
| 知识沉淀 | 读完后想留下什么、用起来 | AI 结构师，帮你构建知识体系 |

## 四、两种阅读模式

### 4.1 模式选择入口

每次点击进入一本书时，展示模式选择界面：

- 显示书籍封面、书名、上次阅读位置
- 两个模式卡片：⚡ 工具式 / 🌊 沉浸式
- 默认选中上次该书使用的模式
- 底部提示："进度和笔记两种模式共享，随时可切换"
- 阅读界面顶部始终可一键切换模式，切换不丢失任何数据与对话记录

---

### 4.2 ⚡ 工具式阅读

**核心理念**：书是原料，问题是导向，输出是目标。

#### 进入流程

**Step 1 · 锚定主体性**

用户填写"本书阅读目标"：
- 提示语：*"这本书你想解决什么问题？你最关心书里的哪个方面？"*
- 自动预填：从**全局用户档案**中提取相关内容，用户可确认或修改
- 提交后：系统合并 `全局档案 + 本书目标` → 生成**阅读上下文**
- 阅读上下文在阅读过程中**随时可修改**（读着读着，目标会变）

**Step 2 · AI 生成专属阅读地图**

- AI 扫描**全书目录 + 全文**（后台异步，不阻塞进入阅读）
- 输出：① 重点阅读章节 ② 可略读章节 ③ 可跳过章节
- 地图呈现在右侧 AI 面板，可点击跳转对应章节

**Step 3 · 进入阅读**

- 阅读界面右侧 AI 面板处于"引导模式"
- 顶部显示当前建议阶段（可拆卸功能，默认开启，可在设置中关闭）

#### 五步工具集（自由调用，无强制顺序）

AI 会根据当前阅读位置智能推荐优先展示哪个工具，但用户可无视推荐自由调用。

**① 列 · 狩猎阅读**
- 悬浮菜单直接展示预生成内容（来自本地 DB，零 token 消耗）：
  - 🌉 跨学科概念联想（如"这与博弈论的囚徒困境有相似结构"）
  - 🔗 旧知识关联（"与您在《X》第N章的[概念]相关"）
- 可选操作：大白话转述 / 加笔记 / 高亮

**② 变 · 三方会谈**
- `批判此观点` — AI 扮演"顶尖反对专家"进行反驳
- `此观点的边界在哪？` — AI 列举观点失效的场景
- `作者为何这样认为？` — AI 分析作者论证逻辑

**③ 缝 · 知识缝合**
- 章节结束时右侧面板出现轻提示卡（用户可忽略）
- `费曼测试` — 用户语音/文字复述，AI 评估准确性并补充遗漏
- `与旧知识连接` — AI 分析两个概念的互补/矛盾/包含关系
- `生成本章知识卡片` — AI 提炼核心要点，存入知识库

**④ 用 · 行动转化**
- 可随时触发（读完部分章节即可，不必等全书）
- 输出：基于**阅读上下文**的个性化行动清单
- 格式：下周/本月可执行的 3-5 个具体步骤
- 可导出为 Markdown 文件

---

### 4.3 🌊 沉浸式阅读

**核心理念**：阅读体验优先，AI 是背景支持。

#### 进入体验
- 直接跳转至上次阅读位置，无引导步骤
- AI 静默带入全局档案上下文
- 右侧 AI 面板默认**收窄**（细条），不占主阅读区

#### 触发 AI 的三种方式

**方式 A · 划词悬浮菜单**

选中文字后浮出菜单，分两区展示：
```
┌─────────────────────────────────────────┐
│  💬 提问  |  ✏️ 高亮 ▼  |  📝 加笔记     │  ← 操作区
├─────────────────────────────────────────┤
│  🌉 这与凯利公式（投资领域）的逻辑相似...  │  ← 预生成区
│  🔗 见《思考快与慢》第8章"损失厌恶"      │    （本地读取，零成本）
└─────────────────────────────────────────┘
```

**方式 B · 快捷键**
- `Cmd + K` — 呼出完整 AI 对话框（带入当前段落作为上下文）
- `Cmd + H` — 快速高亮当前段落
- `Cmd + N` — 快速添加笔记

**方式 C · 展开右侧面板**
- 点击右侧细条 → 展开完整 AI 面板
- 所有工具平铺可用（同工具式，无阶段引导）

#### 系统适时提示（Nudge）

- 默认开启，可在设置中关闭
- 触发场景：读完一章 / 连续阅读 45 分钟
- **关闭方式**：用户继续下滑，或在界面任意位置点击（不自动消失）
- 示例："读完第3章了，要做个费曼测试吗？" / "已连续阅读45分钟，回顾一下高亮笔记？"

## 五、AI 能力体系

### 5.1 Token 成本与模型调度架构：双轨制渐进推断 (Dual-Track Progressive Parsing)  【UPDATE】

**核心原则：坚决避免长文本无底洞式计费，确保上下文精确跟随阅读焦点。**

为了防止百万字长文一次性输入大模型导致 Token 爆炸（高昂的计费和极慢的响应速度），系统放弃了一次性提取全书所有角标的做法，而在后台引擎中解耦了全览与精读：

- **Track 1: 全局大纲引擎（秒级全览）**
  - **触发时机**：新书入库并在首次打开时静默运行。
  - **硬上限防御**：设定 `MAX_GLOBAL_CHARS = 15,000` 字符作为输入红线。
  - **截断采样（Head-and-Tail）**：如果文章超过安全限额，系统放弃中间部分，只会抽取前 10,000 字与后 5,000 字进行拼接。确保即使是一本厚如辞海的史诗巨作，用于概览抽取的输入成本也被绝对锁死在 4500 Token 以内。
  - **产出**：大模型仅返回 150 字精练的【全书/全文提要】，并常挂在右侧 Copilot 面板最顶端。

- **Track 2: 渐进式精读角标追踪引擎（懒加载跟随）**
  - **触发时机**：伴随用户在页面发生的真实滚动动态激活。
  - **动态切割**：将当前视野位置换算成进度率，后台根据进度切取出以当前视口为核心的 **40 个自然段** 构成"当前阅读活动区"（约两到三屏文本）。
  - **推断流**：提取【Track 1 生成的全书大局观摘要】+【这 40 个活动段落】一同发往大模型，密集度地提取 🔑❓💡🌉 解析并回写。这保证了哪怕是你跳读到全书中段，AI 也是结合全局上帝视角为你解答眼前的具体难点。
  - **优势**：真正做到了“看多少，算多少”，完美保护了计费钱包且大幅降低网络吞吐卡顿。
  
【用户主动交互】— 按需 on-demand 调用
  - 用户点击选中划词"提问"，将结合“当前选定段落+全书大局观”精准回答。

### 5.2 段落标注系统

**四种类型**（优先级从高到低）：
- 🔑 蓝色 — 关键论点段落（全书核心主张）
- ❓ 橙色 — 疑难段落（语义密度高或逻辑跳跃）
- 💡 绿色 — 跨书关联点（与用户已读书籍共鸣）
- 🌉 紫色 — 跨学科联想点（可用其他领域类比）

**密度控制规则**（防止视觉噪音）：
- 每段最多显示 1 个角标，同段多种类型取优先级最高者
- 每章角标总数 ≤ 该章段落数的 20%
- 相邻标注段落之间至少间隔 1 段无标注

### 5.3 全局功能

| 功能 | 说明 | 入口 |
|------|------|------|
| 🗺️ 专属阅读地图 | 按用户目标为章节排列权重 | 顶部工具栏 |
| 📊 全书结构分析 | 论点树 / 思维导图 | 顶部工具栏 |
| 🔗 跨书知识图谱 | 用户已读书籍的概念关联网络 | 顶部工具栏 |

## 六、界面设计与空间架构

本系统界面设计遵循**“内容优先，辅助分层”**的核心原则，不仅定义了平面的基本布局框架，更建立了一套严密的 L0 - L3 三维空间拓扑结构，确保组件不割裂、功能不抢夺焦点。

### 6.1 页面基础布局框架

系统提供两种并行的基础平面布局：

#### A. 工具式阅读界面（默认展开 AI）
```text
┌─────────────────────────────────────────────────────────────────┐
│  ← 书库  📗 书名标题区块  ⚡工具式↕ |  [🗺️][📊][🔗][📖]  | ⚙️ │
├─────────────────────────────────────────┬───────────────────────┤
│  🔑  正文内容正文内容正文...             │  【当前情境区】        │
│      正文内容正文内容...                │  第3章：...摘要...    │
│                                         │  本段：🔑 关键论点    │
│  ❓  另一段正文内容内容...              │  ─────────────────── │
│      正文内容...                        │  【工具区】           │
│                                         │  费曼测试             │
│      普通正文普通正文...                │  批判模式  ← 推荐     │
│                                         │  生成知识卡片         │
│  💡  跨书关联段落内容...               │  ─────────────────── │
│      正文内容...                        │  【对话区】           │
│                                         │  💬 AI 对话历史...   │
└─────────────────────────────────────────┴───────────────────────┘
  正文区 65%                               AI 面板 35%
```

#### B. 沉浸式阅读界面（AI 收容态）
```text
┌───────────────────────────────────┬──────┐
│  ← 书库  📗 书名标题区块  🌊沉浸式↕ │ AI ▶ │
├───────────────────────────────────┤      │
│  🔑  正文内容正文内容...           │      │
│      正文内容正文内容...           │      │
│      普通正文普通正文...           │      │
│  ❓  疑难段落内容...              │      │
└───────────────────────────────────┴──────┘
  正文区 90%                    10%（点击唤出）
```

### 6.2 纵向空间拓扑 (V2.0 L0-L3 架构)

在上述二维布局之上，为应对复杂的 AI 弹窗和划词工具，系统在垂直（Z轴）维度做如下分层控制：

**🔴 L0 层：沉浸底板 (The Canvas Space)**
- **元素**：PDF / EPUB 内容渲染器本体。
- **规则**：除了文字本身和底层的系统级高亮块，不允许任何不透明的块状 UI 长期遮挡于此区域之上。

**🟡 L1 层：瞬态意图捕捉器 (划词菜单)**
```text
┌─────────────────────────────────────────┐
│ 🟡 🔵 🟢 | 💬 提问  🧠 解释  📝 笔记  │ 
└─────────────────────────────────────────┘
```
- **视觉分离**：基础高亮颜色组与 AI 复杂操作之间插入 1px 半透明竖线隔离。
- **连击与乐观渲染**：点击颜色后应用“光速乐观渲染”（立刻变色，后台则异步静默保存），但**悬浮菜单绝不消失**。颜色外侧增加白色描边以作状态反馈，支持用户随后连贯点击右边工具触发 AI 解析（“先标黄，随手解析”连招）。

**🟢 L2 层：合体容器模式 (Unified Popover Shell)**
- **消除割裂感**：当在划词菜单点击 AI 动作（如“解释”）时，产生的结果面板**绝不独立弹出**。它必须从上方悬浮条的正下方紧贴着“抽拉/生长”出来，吸附合并为一个**一体化的巨型下拉部件**。
- **像素级融合**：接壤处 `margin: 0`；上方菜单下圆角与下方结果窗上圆角均置为 `0`；外层包裹同一组 `box-shadow` 发光投影。
- **防窒息排版**：面板内的 AI 结果必须符合工业阅读排版：行高 `1.6-1.8` (`leading-relaxed`)，安全内边距 `>= 16px`，段落保留呼吸间距，采用低对比“柔和灰字+深灰底”缓解光晕。

**🔵 L3 层：外边距持久锚点 (Margin Anchors)**
- **挂载 AI 资产隐喻**：对于已生成 AI 深度解读，或记录了本地文字笔记的高亮焦点段落，在文档最右侧安全边距内，悄悄挂载一个微型的发光标识（如 📌 或 💡）。
- **交互通道联动**：Hover 提供浮窗轻度预览，Click 则直接呼唤出最右侧 Copilot 面板，并精确滚屏锚定到相关的历史会话记录。即使关掉 L1/L2 弹窗，优质资产依然有极轻量的定海神针入口。

### 6.3 核心组件全局降噪规范

**❶ 顶栏重设：退居幕后**
- **横向限宽**：顶栏物理宽度严控等于左侧“阅读流容器”宽度，**决不干涉覆盖右侧 AI 面板的 Z-index 及可见性**。
- **抽屉吸顶**：采用靠近吸顶激活的下拉呈现交互（特别在沉浸模式下）。
- **设置收纳**：所有非阅读相关的系统级设置⚙️图标与个人档案，统统转移并折叠至右上角“用户头像”的下拉列表内。

**❷ AI 侧边栏 (Copilot Panel) 的降级与瘦身**
- **消灭裸露异常**：底层的 PDF 解析崩溃日志、ASCII 乱码等，**绝对禁止**前端泄漏回显。系统必须截获异常，统一使用克制的文案降级兜底（“遇到扫描件未知排版，大范围精读受限，但您仍可划出文字对我提问”）。
- **空间折叠**：顶部“全书概要”默认为薄幅面卡片，支持折叠。底部各种庞大的引导大按钮（如“费曼对讲”），重构为高度紧凑的 Chips（圆角标签族），置于最底部聊天输入框的正上方。

## 七、数据架构

### 7.1 双轨存储

| 存储层 | 技术 | 用途 |
|--------|------|------|
| 结构化数据库 | SQLite（本地） | 书籍、进度、笔记、高亮、AI 生成内容、知识卡片 |
| Markdown 文件 | 本地文件系统 | 每本书对应 .md 笔记文件，数据完全可迁移 |

### 7.2 核心数据模型

```
UserProfile（全局用户档案）
  ├── identity: 我是谁、背景、职业
  └── interests: 长期关注的问题和目标

Book（书籍）
  ├── file: 原始文件（EPUB/PDF/TXT）
  ├── metadata: 书名、作者、封面
  ├── readingContext: 本书阅读目标（可随时修改）
  ├── progress: 当前阅读位置
  └── mode: 上次使用的阅读模式

Annotation（段落标注，预生成）
  ├── bookId + paragraphId
  ├── type: KEY | HARD | LINK | BRIDGE
  ├── content: 预生成的对应内容
  └── crossDisciplinary: 跨学科联想文本

Note（笔记 & 高亮）
  ├── bookId + position
  ├── type: highlight | note
  ├── color: yellow | blue | green | red
  └── content: 批注文字

KnowledgeCard（知识卡片）
  ├── sourceBook + sourceChapter
  ├── concept: 核心概念
  ├── summary: AI 提炼的要点
  └── connections: 关联的其他卡片 ID

AIConversation（对话记录）
  ├── bookId + sessionId
  ├── mode: instrumental | immersive
  └── messages: 对话历史（切换模式时保留）
```

## 八、技术架构

### 8.1 技术栈

| 层 | 技术 | 选型理由 |
|----|------|---------|
| 前端 | React + TypeScript + Vite | 生态成熟，热更新快，AI 库丰富 |
| 后端 | Node.js + Express | 轻量，完整文件系统访问，AI 流式支持 |

## 附录：核心设计决策汇总

| 决策项 | 结论 |
|--------|------|
| 支持格式 | EPUB / PDF / TXT |
| 平台 | 桌面端 Web（localhost 本地全栈） |
| 用户规模 | 先个人，后可扩展多人 |
| 技术栈 | React + TypeScript + Vite + Node.js + SQLite |
| AI 模型层 | 多模型 Adapter，用户自配 API Key |
| 数据存储 | SQLite + Markdown 双写 |
| 阅读模式 | 工具式（向导引导）+ 沉浸式（按需调用） |
| 布局 | 左文右 AI，且顶栏采用左侧局部吸顶沉浸态，解耦干扰 |
| Token 策略 | 全景概要采用“首尾采样 15k字硬折断”；细节角标采用“由滚动条驱动的 40段滑动视口按需喂投”。杜绝无意义消耗 |
| 段落角标密度 | ≤20% 段落，每段最多1个，优先级 🔑>❓>💡>🌉 |
| Nudge 关闭 | 用户滚动或点击任意位置关闭（不自动消失） |
| 阅读上下文 | 全局档案 + 本书目标合并，可随时修改 |
| 模式切换 | 随时切换，对话记录完整保留 |
| 工具调用顺序 | 自由，不强制五步顺序 |
| 全局功能入口 | 顶部悬浮工具栏小图标按钮组 |

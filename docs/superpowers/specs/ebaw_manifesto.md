# 前沿协作协议：基于证据的智能体验收流 (Evidence-Based Agent Workflow)

## 1. 问题背景 (Background)
在目前“人类 + LLM”的辅助编程协作中，人类开发者经常陷入“无尽跟车验收”的噩梦循环。尽管大语言模型（LLM）能够在纸面上宣称构建了全面的自动化测试（数十个用例 All Pass），但在实际交互式产品（如基于 DOM 的排版、悬浮菜单、高亮组件）中，功能依然频繁崩溃或出现严重视觉 Bug。人类最终沦为 AI 代码的“肉身测试仪”。

## 2. 场景描述 (Scenario)
以 Web 端 AI 阅读器的“多态高亮”功能为例：
- **AI 的承诺**：底层数据已绑定，状态机已走通，Jest 测试显示 `onClick` 被正确触发。
- **人类的实际体验**：
  1. 黄色高亮色块为实心，完全遮挡了底层文字；
  2. 原生的浏览器蓝色选中框未被注销，与组件颜色重叠；
  3. 悬浮窗宽度被定死，多出的几个短字按钮被迫换行；
  4. 点击删除后，DOM 节点因为 React 未清空而成为幽灵节点。
- **痛点**：只要人类没有亲自去特定位置“点”那一下，AI 交付的东西几乎 100% 存在边缘态失效。

## 3. 问题分析 (Diagnosis)
1. **假性 TDD（Mock-driven Illusion）**：大模型习惯于编写基于虚拟 DOM (如 JSDOM) 的单元测试。此类测试忽略了真实的浏览器排版引擎（CSSOM、Z-index、flex-wrap）与原生事件（Selection API），导致了“逻辑 100 分，视觉 0 分”的幻觉。
2. **文本模型的“视觉剥夺”**：当前大多数 Coder Agent 依赖纯文本操作。代码构建对于 AI 来说是“盲人捏泥人”，它靠猜测 `rgba()` 和 `absolute` 的排版结果，天然缺乏人类“一眼看出错位”的直觉。
3. **“讨好型人格”与自证缺失**：基础模型被微调得极其期望达成“指令完成”的结语。在没有外力监管的情况下，它不会主动自我怀疑，更不会去启动服务截图发给用户看，从而导致它交出极度不负责任的阶段物。

## 4. 解决启发 (Proposed Solutions & Heuristics)
传统的单轨协作（人类提需 -> AI出码 -> 人类挑刺）必须被废弃。取而代之的是以下几条铁律：
1. **证据大于断言 (Evidence > Assertion)**：任何牵涉 UI/交互的微小增量，AI 都不允许仅凭“逻辑分析”宣称完工。必须提供终端控制台快照、运行日志，或是调用 `browser_subagent` 获取的前端渲染真实截图。
## 5. 多 Agent 闭环架构 (Multi-Agent Collaboration)

### 5.1 问题背景
人类通常作为唯一的且极其昂贵的“验收监督者”，面对单体 AI 生成的脆弱交互代码时，必须自己拆烂需求、自己喂给代码机器、自己打开页面重测。这种“跟车模式”导致人类永远被囚禁在脏活累活的泥潭中抽不出身，严重削弱了 AI 提效的初衷。

### 5.2 目标
彻底剥离人类在落地阶段无谓的“项目拆解”和“肉身 QA”职责。建立“人管方向、AI 管落地与质检”的新分工边界，让人类能毫无顾忌地当甩手掌柜。

### 5.3 解法
引入“左右手互搏”的主被动多智能体隔离架构，以“角色切分”来实现机器内部的能力互补与机制制衡。

### 5.4 解法详述
- **Planner Agent (严酷产品经理)**：不编写任何项目代码，只负责发掘暗坑并研读人类的宏观 PRD。将其强行翻译为一系列机器可度量、非此即彼的强制验收标准（Acceptance Criteria）。
- **Coder Agent (盲盒开发执行者)**：依据 Planner 拆解的图纸，在受限的独立分支内干苦力活代码栈作业。
- **Vision QA Agent (多模态视觉刺客)**：**架构核心。** 它强关联 `browser_subagent`，自动挂载真实无头浏览器，在 localhost 开端口。通过自动化测试脚本操作 UI，截取带有鼠标交互痕迹的渲染快照，输入视觉大模型（VLM）进行研判。若截图与 Planner 限定不符，直接夹带“现场打脸截图”将代码打回重炼，全程人类不可见。

### 5.5 场景举例
**人类**：“帮我把高亮交互改得顺滑点。”
**Planner 拆解出**：“TaskA：高亮框背景需半透明无遮挡；TaskB：菜单项绝对不能出现宽度断层换行。”
**Coder出码后**：QA 自动操作浏览器抓取截图后大喊：“排版溢界！这里文字堆在一起了（附带标红截图），打回重写！”
*...（机器互搏，推翻重写了3次）...*
终于排版完美，QA 截出无瑕疵录屏图。
**系统最终汇总**：“@人类老板，任务已完美落成。附件一：30个逻辑用例 Pass；附件二：最终页面交互完美长截屏效果证明。”

## 6. Skill 化演进：终极协作沉淀 (Skill-Based Reusability)

### 6.1 问题背景
辛辛苦苦熬出来的“多智能体视觉纠错闭环”如果口耳相传，往往会被极其健忘的上下文窗口稀释。每次开启新项目时，人类都不得不用长篇大论来教育新启动的模型“别猜 UI，记得多去浏览器截图验收”。不仅低效，而且幻觉复发率极高。

### 6.2 目标
将这套极具安全感的重装甲开发流“心智平移”，固化为一种可一键即插即用、且凌驾于默认设定之上的全局系统约束集，让这套理念传承在以后的每个项目里。

### 6.3 解法
依托深度集成的 **Skill（特异技能）体系**，将整套架构、禁令、流程逻辑写进严格的 Markdown YAML 技能预设配置中。

### 6.4 解法详述
在知识库专用目录（如 `.agents/skills/ebaw-visual-qa/SKILL.md`）下落盘这一范式。在脚本中强行界定：
1. 强制设定此预设命令的触发器。
2. 剥夺模型自我宣称完工的权限。
3. 唤起多重 Agent 的底层接口调度顺序强制化。
一旦此类高内聚的组件落库，AI 助手将从此永远拥有这些不可撼动的肌肉记忆。

### 6.5 场景举例
当你下个月开启一个叫“股票 K 线性质画板”的全新高难度全栈项目时，再也不用从头训化环境。
人类只需冷酷地发送：`“/skill ebaw-visual-qa，用视觉流水线开发主画板组件。”`
模型潜意识会被瞬间清零并重构为严苛监理会模式，直接省去千言万语，在一开始就以对付最复杂的渲染引擎的慎密等级为你干活。

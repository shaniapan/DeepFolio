# 两种阅读模式 · 交互设计草稿 v3

> 状态：大部分已确认，UI 组织方案待最终确认  

---

## 已确认的核心设计原则

- 🤖 **预生成优先**：AI 在后台持续分析，内容随阅读位置主动呈现，而非等用户点击触发
- 🌉 **跨学科联想**：寻找跨领域的结构性相似，让知识连接更奇妙
- 📎 **关联内容直接展示**：旧知识关联等预生成内容直接显示在悬浮菜单内，无需额外点击
- 📚 **全书扫描**：AI 扫描全书目录 + 全文，不局限于前几章
- 🔀 **工具自由调用**：工具式的五步不强制顺序，用户可随时使用任意工具
- 💾 **对话记录持久化**：两种模式切换时，AI 面板对话记录完整保留
- 🎯 **阅读上下文动态修改**：工具式的"阅读目标"在阅读过程中可随时修改（读着读着目标会变）

## AI Token 成本策略

**核心原则：预生成批量处理 + 本地缓存，日常阅读零 AI 调用**

```
【首次打开一本书】（仅此一次的批量调用）
  └── AI 处理全书 → 生成并存入本地 SQLite：
       ├── 全书段落标注（🔑❓💡🌉 角标及对应内容）
       ├── 所有章节摘要
       ├── 跨学科联想（基于全书内容）
       └── 旧知识关联（基于用户已读书库对比）

【日常阅读时】（零 AI 实时调用）
  └── 前端读取本地 DB → 右侧面板显示当前位置对应的预生成内容
      用户感受到"AI 跟着我的阅读进度走"，实际是本地缓存渲染

【用户主动交互时】（按需 on-demand 调用）
  ├── 用户点击"提问"
  ├── 用户触发"批判模式"
  ├── 用户执行"费曼测试"
  └── 用户请求"生成知识卡片"
```

**成本结构**：
- 首次开书：一次较大批量调用（可后台异步，不阻塞进入阅读）
- 日常阅读：几乎零成本（全是本地 DB 读取）
- 用户主动交互：按需少量调用

## 1. 模式选择入口

**1.1 触发时机**
- 每次点击书籍卡片时，弹出模式选择界面
- 显示上次该书使用的模式，作为默认选中项
- 用户可直接点击"继续上次模式"快速进入，或切换

**1.2 模式选择界面包含的信息**
- 书籍封面、书名、上次阅读位置
- 两个模式卡片（⚡ 工具式 / 🌊 沉浸式），各带简要说明
- 底部提示："进度和笔记两种模式共享，随时可切换"

**1.3 模式切换入口**
- 阅读界面顶部导航栏，始终显示当前模式标识，点击可切换
- 切换不丢失任何数据，对话记录保留

## 2. ⚡ 工具式阅读

### 2.1 首次进入一本书（工具式）

**Step 1 · 锚定主体性**
- 弹出"本书阅读目标"填写框
- 提示语：*"这本书你想解决什么问题？你最关心书里的哪个方面？"*
- 自动预填：从全局用户档案中提取相关内容，供用户确认或修改
- 用户提交后，系统合并 全局档案 + 本书目标 → 生成"阅读上下文"
- ⚠️ **阅读上下文可随时在阅读界面内修改**（读着读着，目标会变）

**Step 2 · AI 生成专属阅读地图**
- AI 扫描**全书目录 + 全文**（后台异步，不阻塞进入阅读）
- 输出：① 重点阅读章节 ② 可略读章节 ③ 可跳过章节
- 地图呈现在右侧 AI 面板，可点击跳转

**Step 3 · 进入阅读**
- 阅读界面右侧 AI 面板处于"引导模式"
- 顶部显示当前阶段（可拆卸，默认开启，可在设置中关闭）

---

### 2.2 五步工具集（自由调用，无强制顺序）

> 工具式模式下，五步工具始终可用，AI 会根据当前阅读位置智能推荐优先展示哪个工具，但用户可无视推荐自由调用任意工具。

**① 列（狩猎阅读）**
- 悬浮菜单内直接预生成展示（无需点击）：
  - 🌉 跨学科联想（预生成，来自本地 DB）
  - 🔗 旧知识关联（预生成，来自本地 DB）
- 可选操作：`大白话转述` / `加笔记` / `高亮`

**② 变（三方会谈）**
- 可选操作：
  - `批判此观点` — AI 扮演"顶尖反对专家"
  - `此观点的边界在哪？` — 列举观点失效场景
  - `作者为何这样认为？` — 分析作者论证逻辑

**③ 缝（知识缝合）**
- 章节结束时右侧面板出现轻提示卡（用户可忽略）
- 可选操作：
  - `费曼测试` — 用户语音/文字复述，AI 评估并补充
  - `与旧知识连接` — AI 分析关系（互补/矛盾/包含）
  - `生成本章知识卡片` — 提炼核心要点存入知识库

**④ 用（行动转化）**
- 可随时触发（读完部分章节即可）
- 输出：个性化行动清单（基于当前阅读上下文），可导出 Markdown

## 3. 🌊 沉浸式阅读

### 3.1 进入体验

- 直接跳转至上次阅读位置
- 无引导步骤，AI 静默带入全局档案上下文
- 右侧 AI 面板默认**收窄**（细条，不占主阅读区）

### 3.2 触发 AI 的方式

**方式 A · 划词浮出菜单**
- 上方操作区：`💬 提问` / `✏️ 高亮`（颜色选择）/ `📝 加笔记`
- 下方预生成展示区（来自本地 DB，无需点击即显示）：
  - 🌉 跨学科联想
  - 🔗 旧知识关联

**方式 B · 快捷键**
- `Cmd + K` — 呼出完整 AI 对话框
- `Cmd + H` — 快速高亮当前段落
- `Cmd + N` — 快速添加笔记

**方式 C · 展开右侧面板**
- 点击右侧细条 → 展开完整 AI 面板
- 所有工具平铺可用（同工具式，无阶段引导）

### 3.3 系统适时提示（Nudge）

- 默认开启，可在设置中关闭
- 触发：读完一章 / 连续阅读 45 分钟
- **关闭方式**：用户继续下滑，或在界面任意位置点击（不自动消失）
- 内容示例："读完第3章了，要做个费曼测试吗？" / "建议休息一下，回顾一下高亮笔记？"

## 4. 界面层次组织（三层）

### 层一：文本层（正文区域）

只放**段落级预生成角标**，极度克制：
- 🔑 蓝色角标 — 关键论点段落
- ❓ 橙色角标 — 疑难段落
- 💡 绿色角标 — 跨书关联点
- 🌉 紫色角标 — 跨学科联想点

**密度控制规则**（避免视觉噪音）：
- 每段最多显示 1 个角标，同段多种类型时按优先级取最高：🔑 > ❓ > 💡 > 🌉
- 每章角标总数 ≤ 20% 的段落数量
- 相邻标注段落之间至少间隔 1 段无标注

### 层二：浮动层（按需出现）

- 划词悬浮菜单：操作按钮 + 预生成内容内嵌展示
- Nudge 通知：右下角，用户滚动或点击任意位置后关闭
- 章节结束提示卡：右侧面板弹出，不遮挡正文

### 层三：右侧面板（AI 大脑区）

```
┌──────────────────────┐
│  【当前情境区】        │  ← 读取本地 DB，随滚动位置更新
│  当前章节摘要          │    显示当前段落预生成的洞察
│  本段落预生成角标内容  │
├──────────────────────┤
│  【工具区】           │  ← 常驻工具集
│  工具式：智能推荐排序  │    沉浸式：全部平铺
│  费曼测试 / 批判模式  │    均可自由调用
│  生成卡片 / 行动清单  │
├──────────────────────┤
│  【对话区】           │  ← 用户主动交互历史
│  💬 AI 对话记录       │    切换模式时完整保留
└──────────────────────┘
```

### 全局功能入口（待最终确认）

## 5. AI 功能触发维度分类（已定稿）

### 时间点触发

| 时机 | AI 行为 | 成本 |
|------|---------|------|
| 首次打开书 | 全书批量预生成（段落标注+摘要+联想） | 一次性批量 |
| 章节结束 | 触发 Nudge（读本地缓存，零成本） | 零 |
| 全书读完 | 提示生成行动清单 / 读薄输出 | 按需调用 |

### 全局级功能（通过顶部工具栏访问）

🗺️ 阅读地图 · 📊 全书结构 · 🔗 跨书知识图谱 · 📋 行动清单 · 📖 读薄输出 · 🧩 作者视角

### 段落级功能（文本层角标 + 浮动层悬浮菜单）

- 预生成（本地 DB 读取，零实时成本）：角标内容 / 跨学科联想 / 旧知识关联
- 按需调用（用户主动触发）：批判模式 / 费曼测试 / 提问 / 生成卡片
